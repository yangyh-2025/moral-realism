## 二、核心问题诊断

### 问题1：追随机制结构性缺陷（影响最大）

**现象**:
- 追随关系F1仅0.1897，Recall=0.1125（遗漏了89%的历史追随关系）
- 俄国在50轮中30轮追随英国（100%）→ **历史严重错误**
- 奥斯曼帝国仅1轮追随德国（2%）→ **应为高追随**
- 保加利亚、罗马尼亚、塞尔维亚、荷兰、瑞士、波兰的追随准确率均为0%

**根因分析**（代码层面）:

1. **超级大国永远不能成为追随者**（`simulation_service.py:1314`）:
   ```python
   if power_level == PowerLevelEnum.SUPERPOWER.value:
       return agent_id, None, f"{power_level}保持独立自主，不参与追随"
   ```
   → 英国（强国丙）被判定为超级大国，永远不追随任何人，但代码中它自动成为领导候选人。其他国家只能在领导候选者中选择追随对象。

2. **领导候选者太少**：只有超级大国和大国参与领导竞争（`order_determination.py:168`）。在Project 7中只有3个国家（德/俄/英）是候选者。大多数轮次中只有1-2个候选者，其他国家要么追随这1-2个，要么中立。

3. **追随投票缺乏议题维度**：当前追随决策是"全局性"的，不区分议题。历史上的追随是"单一议题追随"——一个国家可以在军事议题上追随德国，在贸易议题上追随英国。模型中没有这个维度。

4. **60%阈值过高**：19个国家中需要12个追随者才算"有领导者"。历史上很少有国家能获得如此高比例的追随。

### 问题2：行为分布严重失衡

**现象**:
- 行为分布F1仅0.1593
- "威胁"（信息手段）占所有行为的43.3%，每一轮都是最高频行为
- 所有国家（包括小国）的Top-1行为都是"威胁"
- 经济手段（援助、合作）占比9.7%，但从未成为任何国家的主导行为
- 俄国Top-1行为是"交战/使用武力"（40%）→ **1913年和平时期不应如此**

**根因分析**（代码层面）:

1. **"威胁"行为成本过低**：在`action_config`中，"威胁"（ID=9）的国力变化极小，是最低成本的行为。LLM自然倾向选择成本最低的行为。

2. **Prompt中强制行为过多**：`prompt_templates.py`中的17条核心规则大量强调"安全困境"、"预防性威慑"、"冲突升级惯性"等，引导LLM倾向选择威胁和军事行为。

3. **行为价值权重失衡**：各领导类型的行为权重中，军事/威慑类行为有正向加成（如强权型+0.12），而经济合作类行为权重较低。

4. **经济手段缺乏吸引力**："提供援助"和"开展实质性合作"在prompt中没有足够的战略价值描述，LLM认为这些行为收益低。

5. **行为多样性约束执行不力**：虽然规则17要求"行为组合应覆盖多种手段类别"，但实际上LLM连续多轮选择纯威胁行为也未受到惩罚。

### 问题3：领导风格与历史不符

**现象**:
- 德国（霸权型）在16轮中成为领导候选者，但只有3个追随者
- 俄国（强权型）几乎不参与领导竞争
- 英国（王道型）在39轮中成为领导者，追随者最多14个

**根因**:
- 霸权型德国的行为权重过于强调"经济手段"，与历史上的军事扩张行为不符
- 强权型俄国的追随意愿设定为"极低"，导致俄国很少参与领导竞争
- 王道型英国的行为限制过于严格（"禁止执行所有不尊重主权的高烈度对抗行为"），但历史上英国也有殖民战争

## 三、修改方案

### 修改1：重构追随机制（优先级：最高）

**目标**: 实现"单一议题追随"，允许大国有追随行为

#### 1.1 移除超级大国自动独立逻辑

**文件**: `app/services/simulation_service.py`

**当前代码** (line 1314):
```python
if power_level == PowerLevelEnum.SUPERPOWER.value:
    return agent_id, None, f"{power_level}保持独立自主，不参与追随"
```

**修改为**:
```python
# 超级大国也可以在特定议题上追随其他国家
# 不再自动跳过，而是让LLM决定
```

#### 1.2 扩大领导候选者范围

**文件**: `app/core/order_determination.py`

**当前代码** (line 168):
```python
if agent.power_level in [PowerLevelEnum.SUPERPOWER, PowerLevelEnum.GREAT_POWER]:
    candidates.append(agent_id)
```

**修改为**: 允许中等强国也参与领导竞争，或者采用"议题领导"机制——每个议题可以有不同的领导者。

#### 1.3 引入议题维度

**核心思路**: 每轮追随决策应区分议题类型（军事、外交、经济），每个国家可以在不同议题上追随不同领导者。

**实现方案**:
```python
# 追随决策从 {follower_id: leader_id} 改为:
# {follower_id: {military_leader_id, diplomatic_leader_id, economic_leader_id}}
follower_decisions = {
    134: {"military": 130, "diplomatic": 130, "economic": 132},  # 奥匈：军事追随德国，经济可追随英国
    136: {"military": 130, "diplomatic": None, "economic": 130},  # 奥斯曼：军事追随德国
}
```

#### 1.4 降低领导者阈值

**文件**: `app/core/order_determination.py`

**当前**: 60%追随者才算有领导者
**建议**: 降至30-40%，更符合历史上多极体系的实际

### 修改2：重新平衡行为成本（优先级：高）

**文件**: `app/models/action_config.py`（数据库中的action_config表）

#### 2.1 提高"威胁"行为成本

```sql
-- 当前：威胁的国力变化极低
-- 修改：增加威胁行为的国力损耗
UPDATE action_config SET
    initiator_power_change = -0.05,  -- 从几乎为0增加到-0.05
    action_desc = '威胁行为会消耗外交信誉，增加军备负担'
WHERE action_id = 9;  -- 威胁
```

#### 2.2 降低外交/经济行为成本，提高收益

```sql
-- 降低外交合作成本
UPDATE action_config SET
    initiator_power_change = 0.08,  -- 提高收益
    target_power_change = 0.05
WHERE action_id IN (5, 6);  -- 开展外交合作、开展实质性合作

-- 提高援助行为的战略价值
UPDATE action_config SET
    initiator_power_change = 0.06,
    target_power_change = 0.08
WHERE action_id = 7;  -- 提供援助
```

#### 2.3 增加军事行为的国力损耗

```sql
-- 增加军事姿态的成本
UPDATE action_config SET
    initiator_power_change = -0.08
WHERE action_id = 15;  -- 展示军事姿态

-- 增加交战的成本
UPDATE action_config SET
    initiator_power_change = -0.15,
    target_power_change = -0.12
WHERE action_id = 19;  -- 交战
```

### 修改3：优化Prompt模板（优先级：高）

**文件**: `app/core/prompt_templates.py`

#### 3.1 削弱安全困境引导

**当前**: 规则7-8大量强调"安全困境"、"预防性威慑"、"相对收益意识"
**修改**: 减少这些规则的篇幅，增加合作收益的描述

#### 3.2 增加经济手段的战略价值描述

在`CORE_RULES_TEMPLATE`中增加:
```
18. 【经济手段的战略价值】经济合作和援助是长期国力增长的核心途径：
    - 经济合作通过提升pec（能源消耗）和irst（钢铁产量）指标，为长期国力增长奠定基础；
    - 提供援助可以建立战略依赖关系，使受援国在关键议题上倾向追随援助国；
    - 经济手段的成本低于军事手段，且不会引发安全困境的连锁反应；
    - 忽视经济合作的国家将在长期竞争中被经济优先的国家超越。
```

#### 3.3 修改领导类型行为权重

**霸权型**（德国）: 增加军事/威慑类行为的权重，减少纯经济手段的权重
```python
"霸权型": (
    # 修改行为价值权重
    "1. 军事/威慑类行为：额外+0.15战略威慑价值..."  # 从+0.05提高到+0.15
    "2. 经济手段用于建立势力范围：额外+0.1..."  # 新增
)
```

**强权型**（俄国）: 降低追随抵触，增加军事竞争意识
```python
"强权型": (
    # 修改追随决策标准
    "1. 仅考虑候选领导者的军事指标..."  # 改为"优先考虑军事指标，但也考虑战略利益"
)
```

#### 3.4 增加行为多样性惩罚

在`OUTPUT_REQUIREMENTS_TEMPLATE`中增加:
```
8. 【行为多样性硬性约束】每轮选择的行为必须包含至少2种不同手段类别（外交/军事/经济/信息）。
   连续3轮选择单一手段类别的行为组合，将导致国际形象下降，所有行为收益-0.1。
```

### 修改4：调整国力初始值（优先级：中）

**问题**: Project 7中德国CINC=0.2370，英国=0.1872，但德国被分类为"中等强国"而英国是"超级大国"。

**文件**: `app/core/cinc_calculator.py`

**建议**: 检查PowerLevelEnum的划分逻辑，确保CINC最高的国家被正确分类为超级大国。

### 修改5：增加追随关系的历史锚定（优先级：中）

**核心思路**: 在追随决策的prompt中，加入历史锚定信息，引导LLM做出更符合历史的决策。

**文件**: `app/core/prompt_templates.py`

在`build_follower_user_prompt`中增加:
```
【历史背景参考】
- 三国同盟：德国-奥匈帝国-意大利在军事和外交上紧密协调
- 三国协约：英国-法国-俄国在海军和殖民地问题上协调立场
- 巴尔干联盟：塞尔维亚-罗马尼亚-希腊-保加利亚在巴尔干问题上有各自的追随倾向
- 德国军事代表团在奥斯曼帝国，两国军事合作密切
```
