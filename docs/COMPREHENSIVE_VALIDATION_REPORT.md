## 二、跨场景共性问题诊断

### 🔴 核心根因: 威胁(13)是Dominant Strategy

**这是所有问题的根源。**

威胁(13)的成本收益:
- 发起方: +0.10 (国力增加)
- 目标方: -0.20 (国力下降)
- 风险: 无任何负面后果

| 场景 | 威胁占比 | 说明 |
|------|---------|------|
| WWI-id5 | 42.1% (3518/8349) | 接近一半行为都是威胁 |
| WWII | 59.2% (7291/12319) | 超过一半 |
| Cold War | 57.2% (2638/4613) | 超过一半 |

**为什么LLM总选威胁?**
LLM做理性成本收益分析时，威胁(13)是帕累托最优:
- 比外交行为好: 外交合作(5)给双方+0.40, 但威胁只给自己+0.10却损害对手-0.20
- 比军事行为好: 军事行为发起方亏本(-0.10到-0.50), 威胁不亏
- 比经济行为好: 提供援助(7)发起方-0.10, 威胁不亏
- 无风险: 不像军事行为会导致战争升级

### 🔴 问题A: 追随F1低 (Precision极低)

三场景共同表现:
- Recall高(0.69-0.84): 模型能找到该追随的人
- Precision低(0.23-0.39): 大量国家不该追随却追随了

根因: 中立国缺乏"保持中立"的强引导。虽然已添加"中立偏好原则"，但id5是修改前运行的。

### 🔴 问题B: 行为分布严重偏科

| 行为类别 | WWI F1 | WWII F1 | CW F1 | 问题 |
|----------|--------|---------|-------|------|
| 军事手段 | 0.634 | 0.777 | 0.976 | 唯一正常类别 |
| 外交手段 | 0.370 | 0.434 | 0.688 | Recall低 |
| 经济手段 | 0.000 | 0.000 | 0.000 | 完全为零 |
| 信息手段 | 0.000 | 0.000 | 0.000 | 完全为零 |

经济和信息F1=0的原因:
- 经济手段: 没有任何国家以经济为主要行为 (LLM认为经济行为收益低)
- 信息手段: 虽然威胁占比42-59%, 但验证脚本期望的国家都不以信息为主要行为

### 🟡 问题C: 法国/奥匈帝国行为不符历史

| 国家 | 期望 | 实际 | 准确率 |
|------|------|------|--------|
| France (WWI) | 外交 | 军事/信息 | 8% |
| France (WWII) | 外交 | 信息 | 0% |
| Austria-Hungary | 军事 | 信息 | 极低 |

## 四、进一步修改方案 (在已有修改基础上)

### 🔴 最高优先级: 消除威胁(13)的Dominant Strategy地位

**方案1: 调整威胁(13)的power_change** (action_manager.py)

```python
# 当前: 发起方+0.10, 目标方-0.20
# 修改为: 发起方-0.05, 目标方-0.15
# 理由: 威胁应有声誉成本(国际社会对威胁行为的反感)
ActionConfig(
    action_id=13,
    initiator_power_change=-0.05,  # 从+0.10改为-0.05
    target_power_change=-0.15,     # 从-0.20改为-0.15
)
```

**方案2: 添加威胁累积惩罚** (prompt_templates.py)

在CORE_RULES_TEMPLATE中添加:
```
18. 【威胁成本递增】连续使用威胁行为会导致国际信誉下降:
    - 连续2轮威胁: 第3轮起每次威胁额外-0.05国际信誉
    - 连续3轮威胁: 第4轮起每次威胁额外-0.10国际信誉
    - 威胁超过总行为50%: 触发"好战国家"标签, 所有外交行为收益-0.1
```

**方案3: 调整信息手段的power_change结构** (action_manager.py)

```python
# 调查(9): 增加信息获取收益
ActionConfig(action_id=9, initiator_power_change=0.05, target_power_change=-0.10)

# 威胁(13): 增加声誉成本
ActionConfig(action_id=13, initiator_power_change=-0.05, target_power_change=-0.15)
```

### 🔴 高优先级: 补充行为类别多样性硬约束

**方案4: 添加行为配额机制** (prompt_templates.py)

```
19. 【行为配额硬约束】每5轮中:
    - 威胁(13)最多使用2次 (超过则收益递减为0)
    - 至少1次经济手段 (实质性合作或提供援助)
    - 至少1次外交手段 (协商、合作意向、外交合作)
    违反配额的行为仍然可以执行, 但不获得power_change加成。
```
