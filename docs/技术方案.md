## 一、项目概述
### 1.1 业务目标
严格遵循《模型建构_改6.docx》的5项核心假设、智能体规则、决策机制、互动机制、20项标准互动行为集与国际秩序判定逻辑，**以克莱因综合国力方程为唯一国力核算标准**，实现大语言模型(LLM)驱动的基于智能体(ABM)的国际秩序演变仿真系统。

系统定位为**本地单实例、开箱即用、零登录门槛**的学术研究工具，无任何用户注册、登录、权限管理环节，打开浏览器即可直接进入仿真控制台，完整复现学术模型的全部逻辑与仿真能力，满足学术研究的可观测、可复现、可校验要求。

### 1.2 技术目标
1.  模块化分层设计，核心学术逻辑与业务、接口完全解耦，便于调试、扩展与学术迭代；
2.  采用前后端分离一体化架构，后端基于FastAPI构建标准化RESTful接口，前端为单页应用，**无任何登录/认证拦截**，服务启动后直接访问根地址即可使用；
3.  全流程可观测、可断点、可单步调试，完整留存仿真全链路数据、决策日志与国力变化轨迹，本地自动持久化，无需用户配置存储；
4.  高可配置性+零配置启动双模式，**智能体国力配置仅需输入C/E/M/S/W一级指标**，系统自动完成初始综合国力计算与实力层级判定，内置强校验规则；同时预置3套标准学术仿真场景，打开即可一键启动，无需任何手动配置；
5.  100%对齐克莱因国力方程官方计算规则、学术模型层级划分标准与20项GDELT互动行为集，无任何学术逻辑偏差；
6.  内置国力动态更新引擎，严格遵循单次行为国力变动≤10分的硬约束，综合国力随互动行为自动更新，数值完全对齐学术行为集标注；
7.  完整实现学术要求的全量观测指标，支持实力变化、增长率、行为偏好的多维度统计与可视化展示；
8.  彻底移除用户系统、多租户、权限管理相关全部设计，服务单实例运行，所有数据本地存储，无跨设备数据同步，无账号体系，保障开箱即用的使用体验。

### 1.3 核心约束（完全对齐学术模型5项核心假设+行为集硬约束）
完全遵循学术模型的核心设定，技术实现不突破以下边界：
- **假设1：无政府状态假设**：国际体系不存在超越主权国家的全球性中央权威与强制规则执行机制，主权国家是唯一核心行为体，智能体行为完全由自主决策与博弈决定；本模型忽略国际组织、非国家行为体、国内政治机制的影响。
- **假设2：大国主导假设**：国际秩序由大国主导，将国家实力层级划分为超级大国、大国、中等强国、小国四类；仅超级大国/大国可参与国际领导权竞争，是体系博弈的主动发起方；中小国家为被动响应主体，行为聚焦自身生存与利益最大化。
- **假设3：精英决策假设**：智能体完全模拟国家领导集体的决策行为，国家对外决策的制定权始终掌握在少数人构成的领导集体手中，决策直接代表国家行为。
- **假设4：成本收益决策假设**：领导集体所有决策前需完成系统性成本收益权衡，最终优先选择净收益最大化的策略方案；国家利益偏好仅由实力层级决定，与领导集体类型无关，仅昏庸型领导集体可主动偏离国家客观利益。
- **假设5：仅仿真直接机制与客观过程**：核心流程仅覆盖决策机制与互动机制，默认不引入领导人换届、技术突破等额外变量，仅模拟直接机制驱动的国际秩序演变客观过程。
- **国力核算硬约束**：初始综合国力计算严格遵循克莱因方程公式，仿真过程中国力动态更新严格遵循单次行为变动≤10分的硬约束，分值区间、层级判定完全对齐官方标准与学术模型设定。
- **互动行为集硬约束**：所有智能体可执行的行为严格限定在《模型建构_改6.docx》表1的20项标准互动行为集内，禁止生成/执行列表外的任何行为；行为的是否尊重主权属性、发起国/目标国国力变化值完全对齐学术文档标注，硬编码不可修改；行为的发起/响应权限严格遵循实力层级与领导类型约束。

1.  **数据持久层**：负责所有仿真数据、配置数据、日志数据、国力历史数据、互动行为集标准数据的本地持久化存储与读取，是系统的底层数据底座，完全屏蔽底层存储差异，为上层提供统一的数据访问能力；内置克莱因国力指标的字段约束、20项互动行为集的硬编码规则、行为国力影响的硬约束；**无任何用户账号、权限、角色相关的数据表与存储逻辑**。

> ###### 克莱因国力方程官方规则与层级判定标准（硬编码不可修改，100%对齐学术模型）
> 1.  **核心计算公式**：`Pp = (C + E + M) × (S + W)`
>     - 物质要素（C+E+M）：官方满分500分，其中C满分100、E满分200、M满分200，采用基准国比例计分法；
>     - 精神要素（S+W）：官方标准系数为1，S标准值0.5、W标准值0.5，采用专家评定系数法，常规取值区间0~1，支持专家评定0~2；
> 2.  **实力层级判定规则（100%对齐学术模型）**：
>     - 超级大国：`current_total_power ≥ 500` 分（克莱因体系满分基准值）
>     - 大国：`200 ≤ current_total_power < 500` 分
>     - 中等强国：`100 ≤ current_total_power < 200` 分
>     - 小国：`current_total_power < 100` 分
> 3.  **配置规则**：用户仅需输入`c_score、e_score、m_score、s_score、w_score`5个一级初始指标，系统自动完成初始综合国力计算与层级判定，无需输入二级细分指标；输入值超出区间时，系统自动拦截并提示错误；仿真过程中基础指标固定，仅实时综合国力随互动行为动态更新。

    - 增长率分析：支持自定义轮次区间的实力增长率计算，自动统计两类平均增长率：①相同领导类型+相同实力层级的平均实力增长率；②相同领导类型+不同实力层级的平均实力增长率；
    - 行为偏好统计：按国家、实力层级、领导类型三个维度，统计20项互动行为的频次、行为分类占比、主权尊重行为占比、高烈度/低烈度行为占比等指标；
    - 秩序演变统计：全周期国际秩序类型变化时序数据、领导权更迭数据、主权尊重率趋势数据。
6.  **LLM管理服务**：与V1.2版本一致，核心能力：LLM模型配置、API密钥管理、调用参数配置、Prompt模板管理、调用重试与降级，前端可直接配置，无需登录，配置本地持久化。

### 4.4 接口服务层（FastAPI）设计
基于FastAPI实现标准化RESTful接口，所有接口均自带参数校验、异常处理、Swagger在线调试文档，**彻底移除所有用户、认证、权限相关的接口与拦截逻辑**，所有接口可直接调用，适配无登录开箱即用的设计；在V1.2版本基础上，新增预置场景接口、互动行为集查询接口，优化智能体配置接口、统计分析接口，完全对齐修订后的学术模型。

#### 4.4.1 接口通用规范
所有接口统一前缀：`/api/v1`，统一响应体格式，标准异常响应码，支持跨域配置，**无任何token校验、认证拦截、权限控制**，所有接口直接开放调用。

## 五、前后端交互与调试方案
### 5.1 前后端交互核心流程
1.  **启动流程**：后端服务启动后，自动初始化数据库、加载20项互动行为集、加载预置场景；前端访问根地址，直接请求预置场景列表、系统配置，无需任何登录跳转，直接进入首页；
2.  **仿真初始化流程**：用户选择预置场景一键创建项目，或自定义创建项目、配置智能体，前端调用对应接口，后端自动完成参数校验、国力计算、允许行为列表生成，返回项目详情；
3.  **仿真执行流程**：用户启动/单步执行仿真，前端调用流程控制接口，后端异步执行仿真引擎，通过Socket.IO实时向前端推送轮次进度、行为执行记录、国力变化、秩序判定结果，前端实时渲染展示；
4.  **数据查询与统计流程**：用户查看仿真结果、统计指标，前端调用统计分析接口，后端返回结构化数据，前端通过ECharts渲染可视化图表，支持下钻查看详情；
5.  **配置保存流程**：用户修改系统配置、仿真参数，前端调用对应接口，后端本地持久化，无需用户登录，所有配置全局生效。

### 5.2 调试方案
1.  **全链路日志调试**：系统内置全链路日志系统，仿真全流程的引擎执行、LLM调用、行为校验、国力更新、秩序判定均生成详细日志，前端控制台实时展示，支持本地日志文件导出；
2.  **单步断点调试**：支持单轮单步执行，每执行完一个环节自动暂停，可查看当前环节的所有数据、决策详情、校验结果，支持回溯修改后重新执行；
3.  **接口级调试**：FastAPI自带Swagger/Redoc接口调试文档，访问`/docs`或`/redoc`即可直接调试所有接口，无需认证，所有参数内置示例，便于后端模块单独调试；
4.  **仿真数据回溯**：所有仿真数据全量本地存储，支持查看历史任意轮次的完整详情，包括行为记录、决策日志、国力变化、秩序判定结果，便于学术复盘与结果复现。

## 六、部署方案
优化为**开箱即用、零配置启动**的部署方案，支持两种部署模式，均无需任何用户系统、数据库、中间件的额外配置，启动即可使用。

### 6.1 Docker单容器一键部署（推荐）
1.  提供一体化Docker镜像，前后端、数据库、运行环境全部内置，无需额外安装任何依赖；
2.  部署命令：
    ```bash
    # 一键拉取镜像并启动容器
    docker run -d --name intl-order-abm -p 80:80 -v ./data:/app/data intl-order-abm:v1.3
    ```
3.  启动后直接访问`http://localhost`即可使用，无任何登录、配置环节，所有数据持久化到本地`./data`目录，重启容器不丢失数据。

### 6.2 本地原生一键启动部署
1.  针对Windows/macOS/Linux分别提供一键启动脚本，内置Python运行环境、依赖包、前端构建产物，无需用户安装任何软件；
2.  Windows用户双击`start.bat`，macOS/Linux用户执行`./start.sh`，脚本自动启动后端服务、打开浏览器进入系统首页，全程无需人工干预；
3.  所有数据存储在本地软件目录下，卸载直接删除文件夹即可，无残留。

### 6.3 配置说明
- 系统默认使用本地SQLite数据库，无需额外安装配置；
- LLM相关配置首次打开系统时在配置页面填写即可，本地持久化，重启不丢失；
- 支持自定义修改仿真并发数、日志级别、默认场景等参数，均在前端配置页面直接修改。
