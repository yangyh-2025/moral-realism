# ID7 退步根因分析与修复方案

## 核心发现

### 退步数据对比

| 指标 | Project 5 (修改前) | Project 7 (修改后) | 变化 |
|------|-------------------|-------------------|------|
| **追随记录中有领导者** | **725 / 969 (74.8%)** | **144 / 950 (15.2%)** | **-80%** |
| 有追随者的轮次 | 48 / 50 | 41 / 50 | -7轮 |
| 德国Power Level | 超级大国 | 中等强国 | **降级** |
| 追随F1 | 0.6661 | 0.1897 | **-72%** |

### 根因：CINC国力等级阈值变更（commit 48a24a3）

**变更内容**：
```
旧阈值: 超级大国=前10%, 大国=10-30%, 中等强国=30-60%, 小国=后40%
新阈值: 超级大国=前5%,  大国=5-15%,  中等强国=15-35%, 小国=后65%
```

**影响链条**（19个国家）：

```
旧阈值: ceil(19 × 10%) = 2个国家为超级大国 → 英国 + 德国
新阈值: ceil(19 × 5%)  = 1个国家为超级大国 → 仅英国

结果: 德国从超级大国降级为中等强国
```

### 连锁反应机制

```
德国降级为中等强国
    ↓
追随决策中，中等强国必须参与追随投票（不能保持独立）
    ↓
德国被迫在"追随英国"和"追随俄国"之间选择
    ↓
英国（唯一超级大国）自动免除追随义务 → 结构性优势
    ↓
大多数国家选择追随英国（因为英国是唯一稳定的领导者）
    ↓
追随关系单一化（英国一家独大）
    ↓
追随F1暴跌（因为历史上应有多极追随格局）
```

**Project 5 vs Project 7 的领导者分布**：

| 领导者 | Project 5 成为领导者轮数 | Project 7 成为领导者轮数 |
|--------|------------------------|------------------------|
| 英国 | 48轮 | 39轮 |
| 德国 | 30轮 | 16轮 |
| 俄国 | 4轮 | 2轮 |

德国从30轮降至16轮，丧失了近一半的领导力。

## 修复方案

### 方案A：恢复CINC阈值（推荐，最直接）

**文件**: `app/core/cinc_calculator.py`

将阈值恢复为修改前的值：

```python
# 当前值（导致退步）
SUPERPOWER_PERCENTILE = 0.05   # 前5%
GREAT_POWER_PERCENTILE = 0.15  # 前15%
MIDDLE_POWER_PERCENTILE = 0.35 # 前35%

# 恢复为
SUPERPOWER_PERCENTILE = 0.10  # 前10%
GREAT_POWER_PERCENTILE = 0.30  # 前30%
MIDDLE_POWER_PERCENTILE = 0.60 # 前60%
```

**效果**: 德国恢复为超级大国，与英国同等地位，追随格局恢复多极化。

### 方案B：保留新阈值但增加特殊处理（备选）

如果需要保留更严格的等级划分，在追随决策中增加特殊逻辑：

**文件**: `app/services/simulation_service.py`

在 `_generate_follower_decisions` 方法中，将CINC排名前2的国家视为"事实超级大国"，免除追随义务：

```python
# 在 _decide_vote 函数中
# 获取CINC排名
sorted_agents = sorted(agents, key=lambda a: a.get('current_total_power', 0), reverse=True)
top2_ids = {a['agent_id'] for a in sorted_agents[:2]}

if agent_id in top2_ids:
    # CINC前2名保持独立
    return agent_id, None, "CINC排名前2，保持独立自主"
```

### 方案C：综合修复（推荐用于长期）

同时修改以下内容：

1. **恢复CINC阈值**（方案A）
2. **保留行为前置条件校验**（commit 0c15779中的规则15-16，这是好的改进）
3. **保留CINC保护机制**（软下限和变化率上限，防止极端国力波动）
4. **微调追随机制**（保留"追随≠同盟"的弱化版，但不改变阵营惯性惩罚力度）

## 需要保留的改进

commit 0c15779 中有些改进是好的，不应全部回退：

| 改进 | 是否保留 | 原因 |
|------|---------|------|
| 行为前置条件校验（规则15-16） | ✅ 保留 | 防止不合理的"和平时期交战"，提升仿真真实性 |
| 后处理行为过滤 | ✅ 保留 | 兜底机制，防止LLM输出不合规行为 |
| CINC软下限保护（15%） | ✅ 保留 | 防止小国被消灭，保护仿真稳定性 |
| CINC变化率上限（±30%） | ✅ 保留 | 防止国力暴涨暴跌 |
| Scale factor调低 | ⚠️ 需评估 | 降低单次冲击幅度，但可能过于保守 |
| 追随机制"追随≠同盟" | ⚠️ 需评估 | 概念正确，但可能导致追随过于松散 |
| **CINC阈值变更** | ❌ **必须回退** | **这是退步的根因** |
