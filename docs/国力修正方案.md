# 国力变化幅度过大：三层防护修正方案

> **状态：已实施**（2026-05-25）
> 
> 实施文件：
> - `app/core/cinc_power_update.py` — 三层防护引擎
> - `app/services/agent_service.py` — 初始CINC计算修复
> - `app/services/scene_service.py` — 批量添加后统一计算

## 问题诊断

CINC比例结构 + 缩放因子 + 指标下限为0 的三重叠加导致：
- 一次"交战"对目标方军事支出冲击：`-0.9 × 2000 × 0.5 = -900`（千美元）
- 大国指标被数十轮持续削至接近0 → CINC贡献项趋近于0
- 小国即使什么都没做，也因"分母缩小"而CINC暴涨

## 解决方案：三层防护机制

修改 `app/core/cinc_power_update.py`，在三个层面施加约束：

### 第一层：降低军事缩放因子（削弱单次冲击）

```python
# 修改前
DEFAULT_INDICATOR_SCALE_FACTORS: Dict[str, float] = {
    "milex": 2000.0,   # 千级
    "milper": 200.0,   # 百级
    "irst": 1000.0,    # 千级
    "pec": 2000.0,     # 千级
    "tpop": 20000.0,   # 万级
    "upop": 10000.0,   # 万级
}

# 修改后：军事指标缩放因子减半，降低军事行为的破坏力
DEFAULT_INDICATOR_SCALE_FACTORS: Dict[str, float] = {
    "milex": 800.0,    # 从2000降至800（降低60%）
    "milper": 80.0,    # 从200降至80（降低60%）
    "irst": 800.0,     # 从1000降至800（降低20%，工业指标衰减较慢）
    "pec": 1600.0,     # 从2000降至1600（降低20%）
    "tpop": 20000.0,   # 人口指标不变（战争对人口的直接影响较小）
    "upop": 10000.0,   # 城市人口不变
}
```

**效果**：一次"交战"对目标方军事支出的冲击从 `-900` 降至 `-360`，对发起方从 `-300` 降至 `-120`。 military手段仍具破坏性，但不再数轮内将大国军事指标削光。

### 第二层：引入指标软下限（防止指标被削至0）

在 `CINCPowerUpdateEngine` 的 `update_round` 方法中，修改指标更新逻辑：

```python
# 在 __init__ 中增加软下限比例参数
self.indicator_floor_ratio = 0.15  # 指标不能低于初始值的15%

# 在 update_round 方法中，修改 Step 2 的指标应用逻辑

# 修改前：
# for aid, (change, _) in agent_changes.items():
#     agent = self._agents[aid]
#     for ind in CINC_INDICATORS:
#         delta = getattr(change, f"{ind}_delta", 0.0)
#         new_val = agent[ind] + delta
#         agent[ind] = max(0.0, new_val)  # 硬下限为0

# 修改后：引入软下限 + 非线性衰减
def _apply_indicator_change_with_floor(
    self,
    agent: Dict[str, Any],
    change: IndicatorChange,
) -> None:
    """
    应用指标变化，引入软下限保护。
    当指标接近下限时，变化量非线性衰减，防止被削至0。
    """
    # 获取初始值（从agent_config加载时保存）
    initial_indicators = agent.get("initial_indicators", {})
    if not initial_indicators:
        # 首次运行时保存初始值
        initial_indicators = {ind: agent[ind] for ind in CINC_INDICATORS}
        agent["initial_indicators"] = initial_indicators

    for ind in CINC_INDICATORS:
        current = agent[ind]
        initial = initial_indicators.get(ind, current)
        delta = getattr(change, f"{ind}_delta", 0.0)

        # 计算软下限：初始值的15%（但不低于原始数据下限的50%）
        floor = max(initial * self.indicator_floor_ratio, 50.0 if ind in ["milex", "pec"] else 5.0)

        if delta < 0:  # 只有负向变化需要衰减保护
            # 当指标接近下限时，变化量非线性衰减
            # 公式：effective_delta = delta * ((current - floor) / (initial * 0.5)) ** 2
            # 当 current = floor 时，effective_delta = 0（完全停止下降）
            # 当 current = initial 时，effective_delta = delta（全额应用）
            if current > floor:
                # 衰减系数：距离下限越远，衰减越小
                distance_ratio = (current - floor) / max(initial - floor, 1.0)
                attenuation = min(1.0, distance_ratio ** 0.5)  # 平方根衰减，越接近下限越敏感
                effective_delta = delta * attenuation
            else:
                effective_delta = 0.0  # 已达下限，不再下降
        else:
            effective_delta = delta  # 正向变化不衰减

        new_val = current + effective_delta
        agent[ind] = max(floor * 0.5, new_val)  # 硬保底线为软下限的50%
```

**效果**：当德国milex从初始值（假设10000）降至约1500（15%软下限）时，进一步交战对其milex的影响将急剧衰减至接近0。这模拟了**国家即使战败也不会把所有军队和装备全部损失殆尽**的现实——总有残余力量、后备产能和恢复能力。

### 第三层：引入CINC变化率上限（防止极端跳变）

在 `update_round` 方法的 Step 3-4 之间插入截断逻辑：

```python
# 在全局重算CINC之后，应用CINC变化率上限
def _clamp_cinc_change(
    self,
    old_cincs: Dict[int, float],
    new_cincs: Dict[int, float],
    max_change_rate: float = 0.30,  # 每轮CINC变化上限为±30%
) -> Dict[int, float]:
    """
    对CINC变化率进行上限截断，防止极端跳变。
    """
    clamped = {}
    for aid, new_cinc in new_cincs.items():
        old_cinc = old_cincs.get(aid, new_cinc)
        if old_cinc <= 0:
            clamped[aid] = new_cinc
            continue

        # 计算变化率
        change_rate = (new_cinc - old_cinc) / old_cinc

        # 截断超限变化
        if change_rate > max_change_rate:
            # 正向变化过大（小国被动膨胀）
            clamped[aid] = old_cinc * (1 + max_change_rate)
            if self.enable_logging:
                logger.warning(
                    f"国家{self._agents[aid]['agent_name']} CINC正向变化"
                    f"{change_rate:+.1%} 超过上限{max_change_rate:.1%}，"
                    f"截断至 {clamped[aid]:.6f}"
                )
        elif change_rate < -max_change_rate:
            # 负向变化过大（大国暴跌）
            clamped[aid] = old_cinc * (1 - max_change_rate)
            if self.enable_logging:
                logger.warning(
                    f"国家{self._agents[aid]['agent_name']} CINC负向变化"
                    f"{change_rate:+.1%} 超过上限-{max_change_rate:.1%}，"
                    f"截断至 {clamped[aid]:.6f}"
                )
        else:
            clamped[aid] = new_cinc

    return clamped
```

在 `update_round` 中的调用位置：

```python
# Step 3: 全局重算CINC
agents_indicators = [...]
new_cincs = CINCCalculator.calculate_all_cincs(agents_indicators)

# === 新增：第三层防护 ===
new_cincs = self._clamp_cinc_change(old_cincs, new_cincs, max_change_rate=0.30)

# Step 4: 全局重算层级（使用截断后的CINC）
new_levels = CINCCalculator.determine_all_power_levels(new_cincs)
```

**效果**：即使CINC比例结构产生极端变化，单轮变化也被限制在±30%以内。德国从CINC=1.0暴跌需要至少数轮（1.0 → 0.7 → 0.49 → 0.34 → ...），而非一轮内腰斩。

## 三层防护的预期效果

| 防护层 | 机制 | 对德国CINC暴跌的干预 |
|:---|:---|:---|
| 第一层 | 缩放因子降低 | 单次交战对milex冲击从-900降至-360 |
| 第二层 | 指标软下限 | milex降至初始值15%后不再大幅下降 |
| 第三层 | CINC变化率上限 | 每轮CINC最多下降30%，50轮从1.0降至约0.001（若持续） |

**综合效果**：在50轮仿真中，德国CINC的下降将从当前的-94.1%收窄至约-70%~-80%（具体取决于行为强度），小国被动膨胀的幅度也将相应缩小。系统仍保留"军事冲突削弱国力"的核心机制，但避免了"数轮削光"的非现实极端。

## 参数可调性

三层防护的参数均可通过系统配置动态调整，无需修改代码：

```python
# 在 system_config 表中增加以下配置项：
{
    "cinc_scale_factors": {
        "milex": 800, "milper": 80, "irst": 800,
        "pec": 1600, "tpop": 20000, "upop": 10000
    },
    "indicator_floor_ratio": 0.15,      # 指标软下限比例（默认15%）
    "cinc_max_change_rate": 0.30,       # CINC单轮变化上限（默认30%）
    "indicator_attenuation_exponent": 0.5  # 衰减曲线指数（默认0.5=平方根）
}
```

研究者可通过调整这些参数进行敏感性分析，找到最符合历史现实的参数区间。
